## Feedback reform

User saw 90 `clip_feedback` rows with `actual_viral_score: 0.0` despite never submitting feedback. Root cause: `PipelineOrchestrator.saveClipsFromScoredSegments` auto-called `saveFeedbackSnapshot` for every scored clip.

- `PipelineOrchestrator.java` — auto-snapshot call removed; `FeedbackService.saveFeedbackSnapshot` method deleted.
- `ai-pipeline/feedback.py` — new `calculate_viral_score(..., hours_since_post, followers=0)`. Formula: velocity × 0.35 + engagement × 0.25 + save_rate × 0.20 + comment_rate × 0.20. Velocity = `log10(views_per_day) / 6`. `--followers` kept for back-compat, ignored.
- `FeedbackService.submitTikTokMetrics` — takes `postedAtIso`, validates parse + not-future, computes `hoursSincePost = max((now - postedAt)/3_600_000.0, 0.1)`, passes `--hours-since-post`. Feature snapshot captured at submit time via new `serializeFeatures(clipId)` reading current `ClipScore` row.
- `FeedbackController.java` — body is `Map<String,Object>`; `postedAt` required (400 on missing/blank).
- `frontend/src/lib/api.ts` — `submitFeedback` signature adds `postedAt: string` (ISO-8601).
- `frontend/src/app/page.tsx` — ClipCard form: `<input type="datetime-local">` with label "Tanggal & jam upload ke TikTok"; `handleSubmitFeedback` validates parseable + not-future; sends `postedDate.toISOString()`. `feedbackError` state for inline validation.
- `clip_feedback` table truncated.
